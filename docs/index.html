§02

Three parts. No platform.

SYSTEM SCHEMATIC

Files you own

Context packs, policies, playbooks, run records, audit logs — plain markdown, YAML, and JSONL under ~/.aos. Readable by any agent, from any provider, forever.

~/.aos/projects/<id>/
├── context/pack.md      # the brief
├── context/decisions.md # why, forever
├── policy.yaml          # the gates
├── learnings.md         # compounds
├── playbooks/
└── runs/<date>-<ticket>/
    ├── ticket.md  plan.md
    ├── verification.md  outcome.md
    └── audit.jsonl      # everything

Enforced, not remembered

Hooks load context into every session, gate risky commands against policy, and write the audit trail — automatically. The /aos-ticket pipeline runs work from intake to verified outcome.

One window, local only

Decision queue, run states, verification verdicts, token economics, and your leverage ratio. Binds to 127.0.0.1. Nothing leaves the machine.

§03

The six layers every operator needs

OPERATIONS DOCTRINE
L1

Verification

Deterministic contracts (tests, lint, checklists) that fail loudly, plus an adversarial reviewer whose only job is to refute the work before you see it.

metric: % of runs clean on first pass — your leverage ratio
L2

Memory

Per-project context packs, decision logs, and learnings — loaded into every new session automatically. Agents stop waking up new.

the re-briefing tax → zero
L3

Mission control

Every run, every state, every project in one queue. What's running, what failed, what's blocked on you — batched, not forty interruptions.

your attention, firewalled
L4

Guardrails

Policy as config: forbidden actions blocked, gated actions held for approval, everything else flows. Enforced by hooks, not promises.

blast radius: bounded
L5

Economics

Token spend per run, per project, per session — recorded automatically. Know which work is cheap leverage and what's quietly expensive.

cost per outcome, visible
L6

Learning loop

Every run ends by distilling what was learned into project memory. Repeated procedures become playbooks. Run #30 is faster than run #3.

compounding, by default

§06

Operating principles

NON-NEGOTIABLE

Files over platforms.

Markdown, YAML, JSONL in your home directory. git init ~/.aos and your entire operation has version history. No accounts. No lock-in.

Enforced beats remembered.

Guardrails and audit live in hooks that fire on every action — not in prompts an agent might forget under context pressure.

Never self-certify.

The agent that did the work doesn't get to declare it correct. Contracts fail loudly; a paid skeptic tries to refute everything.

Every layer works standalone.

Hooks alone are worth installing — context, gates, and audit with zero workflow change. Adopt the rest when the pain arrives.

Local first, local only.

The console binds to 127.0.0.1. Your clients' context, your policies, your audit trail — nothing leaves the machine until you ship it.
